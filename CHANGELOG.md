# 版本记录

## [1.2.0] - 2026-04-16

### 重大变更

- **三问法升级为五问电梯测试**：从3个宽泛问题+多轮追问，改为5个精准问题+每问最多追问1次
- **2分钟原则**：整个对话控制在5-8轮问答内，说不清楚=不懂
- 问题拆分：
  - "为什么好" 拆为 Q1（靠什么赚钱）+ Q2（护城河是什么）
  - "为什么现在买" 精简为 Q4（为什么是现在）
  - 隐性追问整合为 Q5（买错怎么办）

### 新增

- 五问核心框架表：每个问题明确"测什么/好回答标准/差回答信号"
- 每问追问话术模板（只在需要时使用）
- 追问规则：每个问题最多追问1次，不纠缠
- 2分钟时间分配建议

### 修改

- 报告模板：三问改为五问，隐性追问改为认知偏差检测
- 三问法指南全面重写为五问电梯测试指南
- Pitfalls更新：从"至少追问2-3轮"改为"最多追问1次"

## [1.1.0] - 2026-04-15

### 新增

- **全面股票分析模块**：AI 在验证期独立完成专业级股票分析（商业模式、财务健康、估值、竞争、催化与风险），作为"正确答案"与用户认知对照
- **持续关注指标模块**：区分"决定性指标"（变了就推翻逻辑）和"辅助性指标"（影响估值），包含当前值、关注阈值、监测频率、数据来源
- **指标监测日历**：按时间列出关键数据发布节点
- 报告从9模块升级为11模块

### 优化

- 验证期升级：从"纯验证"到"验证+全面分析"双轨并行
- 投资建议增强：不同认知定级的建议中增加"对照正确答案找差距"和"关注决定性指标"的引导
- 风险提示增加：提醒"全面股票分析"基于截止日期数据，市场可能随时变化
- 版本号升级至 1.1.0

## [1.0.1] - 2026-04-15

### 优化

- 新增"一问一答原则"：每次只问一个问题，确保用户有充分的回答空间
- 强化追问策略：明确禁止一次性抛出多个问题
- 优化 Pitfalls：添加第10条关于一问一答的强制规范

## [1.0.0] - 2026-04-15

### 新增

- 初始版本发布
- 四阶段工作流：采集期 → 结束信号 → 验证期 → 报告期
- 三问法框架：为什么好 / 为什么便宜 / 为什么现在买
- 隐性追问：退出纪律 / 仓位管理 / 认知偏差检测
- 自适应追问力度：根据用户回答质量动态调整
- 四维度认知评分：商业模式理解 / 财务认知 / 行业认知 / 风险认知
- 三级认知定级：了解 → 熟悉 → 精通
- 一次性全网交叉验证：neodata + tushare + web_search
- 认知评价报告模板（9模块）
- 跨平台兼容：WorkBuddy / Trae / Claude Code / OpenCode / OpenClaw
- 一键更新脚本：update.ps1 / update.sh
